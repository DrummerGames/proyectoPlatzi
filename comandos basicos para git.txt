git status : para verificar el estado de la modificacion del archivo o del documento en cuestion.

git add <archivo> | <.> : para agregar al STAGE (estado previo al commit) los archivos a comitear.

git commit -m " <mensaje del commit>" : en este punto se afirma la version que se quiere guardar.

git push <origin> <branch o rama> : para subir las versiones al branch en GITHUB.⏫

git checkout <branch | commit> : te envia a la version del commit o el branch en el que tu vas a trabajar.

git checkout -b <branch> : te crea un branch con el nombre que le asignes y te envia a ese branch a trabajar.

git pull <origin> : Trae los datos desde GitHub. ⏬

git fetch <origin> : Trae TODA la METADATA desde Github.
